# 基因突变教学模拟器 V1 详细方案

更新时间：2026-04-22

## 1. 项目定位

本项目面向中国高中生，目标是做一个可在 iPad、Android 平板和电脑浏览器中使用的交互式教学工具，帮助学生理解：

- 基因突变的基本类型与后果
- 镰状细胞贫血的典型案例
- DNA、mRNA、密码子、氨基酸和性状之间的联系
- 自然选择如何影响种群中的基因频率
- 在实验探究过程中，如何提出问题、观察结果并做出生物学解释

产品形态定位为：

- 一个以网页为载体的互动教学应用
- 模拟器主体支持离线使用
- AI 聊天助手仅在联网时可用
- 优先适配课堂教学和学生自主探究场景

## 2. 核心用户与使用场景

### 2.1 目标用户

- 中国高中生物教师
- 中国高中学生

### 2.2 主要使用场景

- 教师在课堂大屏上进行案例演示
- 学生使用 iPad 或 Android 平板进行分组探究
- 学生使用电脑在课后进行复习和补充练习
- 网络不稳定环境下，继续使用核心模拟器功能

## 3. 产品目标

### 3.1 教学目标

- 让学生理解替换、增添、缺失三类突变的差异
- 让学生理解某些突变会导致沉默突变、错义突变、无义突变或移码突变
- 让学生通过镰状细胞贫血案例建立“基因变化 -> 蛋白质变化 -> 性状变化”的因果链
- 让学生理解自然选择会改变种群中的基因频率，而不仅仅作用于个体

### 3.2 产品目标

- 界面清晰，适合高中生快速上手
- 交互简单，触控友好
- 教学内容准确但不过度学术化
- 支持联网和断网两种状态下的合理降级
- 预留后续扩展更多遗传案例和课堂任务的空间

## 4. 设计原则

- 教学优先：所有交互都应服务于概念理解，不追求炫技
- 可观察：每一步操作都应看到结果变化
- 可解释：不仅显示结果，还要说明原因
- 触控优先：平板端点按体验优先于鼠标悬停体验
- 渐进展开：先给典型案例，再给自由探究
- 稳定优先：断网时模拟器主体仍能运行
- 中国课堂友好：默认中文表达，术语简明，避免过重英文界面

## 5. V1 功能范围

### 5.1 模块 A：镰状细胞贫血典型案例

目标：作为学生进入应用后的第一个重点案例，帮助其理解点突变与表型改变的关系。

计划内容：

- 展示正常红细胞与镰状红细胞的形态对比
- 展示相关基因片段的正常序列与突变序列
- 展示 DNA、mRNA、密码子和氨基酸的逐步变化
- 用简洁语言解释为什么蛋白质性质变化会影响红细胞形态
- 在模块结尾引出自然选择与群体遗传问题

建议交互：

- 分步切换“正常 -> 突变 -> 转录 -> 翻译 -> 性状”
- 关键变化位置高亮
- 每一步配有一句解释和一个“再简单一点”的按钮

### 5.2 模块 B：自定义突变实验台

目标：让学生主动操作不同突变并观察后果。

V1 支持的操作：

- 替换某一位碱基
- 插入一位或一段碱基
- 删除一位或一段碱基
- 重置到初始序列

每次操作后同步显示：

- 原始 DNA 序列
- 突变后 DNA 序列
- 转录后的 mRNA
- 翻译后的氨基酸序列
- 突变分类结果
- 一段面向高中生的解释

V1 建议简化策略：

- 默认给出编码链视角，避免初版就让学生陷入模板链方向问题
- 保留“教学说明”解释这是为了简化课堂理解
- 后续高级模式再补充模板链、阅读框等更严格内容

### 5.3 模块 C：自然选择与基因频率模拟

目标：让学生理解自然选择对种群等位基因频率的影响。

V1 输入参数：

- 初始等位基因频率
- 种群规模
- 世代数
- 环境条件开关，例如是否存在疟疾压力
- 不同基因型的相对生存率

V1 输出内容：

- 每代等位基因频率折线图
- 每代基因型频率柱状图
- 当前条件下的文字解释
- 一个“课堂问题提示”区域，引导学生说出观察到的趋势

V1 定位：

- 教学模拟，不做复杂群体遗传高级计算器
- 参数数量控制在可课堂使用的范围内

### 5.4 模块 D：AI 生物问答助手

目标：作为课堂探究助手，帮助学生理解当前实验和高中生物概念。

功能边界：

- 只回答生物相关问题
- 优先回答中国高中生物课程相关内容
- 优先结合当前模拟上下文作答
- 非生物问题礼貌拒答
- 不提供医疗诊断，不替代教师评价

建议能力：

- 解释当前实验结果
- 帮助学生理解术语
- 引导式提问，如“你可以先比较突变前后的密码子”
- 用更简单的话重述答案
- 用“高考答题风格”简洁归纳

## 6. 页面结构规划

### 6.1 首页

内容建议：

- 项目标题与一句话介绍
- 进入案例模式
- 进入自由探究模式
- 进入自然选择模拟
- 查看使用说明
- 网络与离线状态提示

### 6.2 案例模式页面

布局建议：

- 主内容区展示案例步骤和图示
- 右侧或底部展示术语说明
- 平板端采用分段卡片式布局

### 6.3 自定义突变实验台页面

布局建议：

- 上部为 DNA 操作区
- 中部为转录与翻译结果区
- 下部为解释卡片区
- AI 聊天入口采用抽屉或侧栏

### 6.4 自然选择页面

布局建议：

- 左侧参数控制区
- 右侧图表展示区
- 下部为文字结论和课堂问题

### 6.5 AI 聊天窗口

电脑端：

- 固定右侧可收起侧栏

平板端：

- 右下角按钮打开抽屉面板

聊天窗口中需要提供：

- 示例问题快捷按钮
- 当前网络状态提示
- 当前问题范围提示
- “结合当前实验解释”快捷操作

## 7. 多设备适配原则

### 7.1 通用原则

- 所有核心操作支持触控
- 不依赖 hover 展示关键内容
- 按钮和可点击区域尺寸适合手指点击
- 文本字号和行距适合平板阅读

### 7.2 iPad 适配

- 重点兼容 Safari
- 横屏优先设计，竖屏可用
- 减少输入框数量，避免键盘弹起遮挡核心界面
- 支持添加到主屏幕后的类 App 体验

### 7.3 Android 平板适配

- 重点兼容 Chrome
- 注意不同分辨率下的布局伸缩
- 控制动画和图表渲染开销，避免中端设备卡顿

### 7.4 电脑端适配

- 显示更完整的信息密度
- 提供更宽的图表和双栏布局
- AI 侧栏默认可见或可快速展开

## 8. 离线与联网策略

### 8.1 总体原则

- 模拟器主体支持离线使用
- AI 功能仅在联网时可用
- 离线状态下应有明确提示，而不是静默失败

### 8.2 离线支持方式

计划将应用做成 PWA，首次联网访问后缓存核心资源。

需要预缓存的内容：

- 首页与主要模块页面
- 核心前端脚本
- 主要样式文件
- 案例数据 JSON
- 基础图示和本地图片资源

离线后可继续使用的功能：

- 镰状细胞贫血案例浏览
- 自定义突变实验
- 自然选择模拟

离线后不可使用的功能：

- AI 聊天
- 任何实时联网请求

### 8.3 用户使用方式

- 首次需联网访问网站
- 访问后核心资源缓存到设备本地
- 后续断网时，学生仍通过同一网址或安装后的主屏幕图标进入应用

## 9. 技术方案

### 9.1 前端

建议技术栈：

- Next.js
- TypeScript
- React
- PWA 支持

理由：

- 便于同时开发页面和服务端接口
- 组件化结构清晰
- 适合后续部署到 Vercel
- 方便做响应式布局和后续扩展

### 9.2 后端接口

采用 Next.js API Route 或 Route Handler，统一暴露：

- `/api/chat`

前端永远只调用自己的接口，不直接请求模型厂商。

### 9.3 AI 供应商适配层

计划支持：

- 阿里云百炼
- Kimi API

后端内部使用统一 provider 抽象：

- `provider = bailian`
- `provider = kimi`

建议策略：

- 百炼为主通道
- Kimi 为备选通道
- 主通道失败时可切换到备通道

### 9.4 数据组织

建议将教学案例与说明文本本地化为 JSON 或 TS 数据文件，例如：

- 镰状细胞贫血案例数据
- 示例突变序列
- 自然选择默认参数模板
- AI 示例问题列表

### 9.5 图表与可视化

可选用轻量图表方案展示频率变化趋势，原则是：

- 移动端和中端设备上渲染稳定
- 不依赖大型复杂图形库

## 10. AI 限定与安全策略

### 10.1 系统提示词原则

- 只回答生物相关问题
- 优先高中生物课程范围
- 回答尽量清晰、准确、简洁
- 优先结合当前实验界面上下文
- 对非生物问题直接拒答

### 10.2 服务端拦截策略

在调用模型前增加问题分类：

- 明显非生物问题：直接返回固定拒答语
- 明显生物问题：正常调用
- 边界问题：提示聚焦到生物课程相关内容

### 10.3 上下文增强

发送给 AI 的上下文可包括：

- 当前模块名称
- 当前突变前后序列
- 当前密码子和氨基酸变化
- 当前是否为移码突变
- 当前自然选择参数和结果摘要

## 11. 教学内容表现建议

### 11.1 语言风格

- 使用简洁中文
- 避免大学教材式过度术语化表达
- 适度保留考试高频术语
- 解释中多用“因为……所以……”

### 11.2 结果说明卡片

每次实验结果建议统一采用固定结构：

- 发生了什么变化
- 为什么会这样
- 对蛋白质或性状可能有什么影响
- 你可以进一步观察什么

### 11.3 课堂友好设计

- 提供示例问题按钮
- 提供“再简单一点”按钮
- 提供“教师讲解版”与“学生探究版”两种入口

## 14. 风险与注意事项

### 14.1 网络风险

- Vercel 在中国大陆网络环境下可能存在访问波动
- 自定义域名通常会比默认域名更稳，但不保证所有网络环境都稳定

### 14.2 AI 风险

- AI 需要联网
- 不同模型回答风格可能不完全一致
- 必须有主题限制与拒答策略，避免偏题

### 14.3 教学风险

- 如果初版信息太多，学生会被界面复杂度劝退
- 如果表达太简化，教师可能担心不够严谨
- 需要在“课堂易懂”和“科学准确”之间平衡
